Portal do desenvolvedor
Visão geral
Este documento lhe dará uma compreensão básica do Portal do Desenvolvedor da Riot Games. Ele foi projetado para ajudar você a começar a explorar e desenvolver suas próprias ferramentas e produtos para impactar positivamente a experiência dos jogadores dos jogos da Riot. Mal podemos esperar para ver o que você faz!

Primeiros passos
Antes de começar a aproveitar o Portal do Desenvolvedor, você deve login com o seu Jogos de motim conta. Depois de fazer isso, uma conta do Portal do Desenvolvedor será criada para você! Esta ação também gera uma chave básica de API de desenvolvimento associada à sua conta. Falaremos sobre como você pode interagir com a Riot para aprimorar essa chave mais adiante nesta documentação.

Esta conta lhe dá a opção de registrar sua proposta de produto com a equipe de Relações com Desenvolvedores da Riot. Depois de registrar seu produto, você pode se comunicar com nossa equipe para aprovações, maior acesso e muito mais.

Registro de produto
Se você está aqui, provavelmente está pensando em criar um aplicativo ou site legal para um jogo da Riot. Para simplificar, chamaremos amplamente essas coisas de "produtos". Conforme mencionado acima, depois de fazer login no Portal do Desenvolvedor, você poderá registrar sua proposta de produto. Vamos analisar o que isso implica.

Processo de Aplicação
Em primeiro lugar, espera-se que todos os proprietários de produtos (desenvolvedores) tenham lido e compreendido as nossas políticas. Você pode encontrá-los na barra de navegação no topo desta página. Leia-os. O não cumprimento dessas políticas pode levar a ações punitivas, portanto, certifique-se de entendê-las antes de avançar muito.

Mesmo que seu produto não use a API da Riot Games, será benéfico registrá-lo. O registro permite que a Riot saiba que você está lá fora, fazendo ótimas coisas. Se sabemos que você está por aí, podemos mantê-lo informado sobre oportunidades que podem ser benéficas para você e seu(s) produto(s). Além disso, você terá um caminho para nos informar como o Riot pode ajudar melhor você e outros desenvolvedores.

Então, como você se registra?

É simples. Volte para a página principal do Portal do Desenvolvedor e clique em Registrar produto botão. A partir daí, você precisará decidir se isso é para um produto de maior escala ou um projeto pessoal. A partir daí, pediremos que você preencha um formulário on-line que reúna alguns detalhes importantes sobre seu produto. Por fim, pediremos que você verifique seu produto, apenas para ter certeza de que está realmente o desenvolvedor!

Depois de concluir essas etapas, nossa equipe de Relações com Desenvolvedores analisará sua proposta. Se tudo estiver em ordem, aprovaremos seu produto e você estará a caminho de desbloquear limites de tarifas adicionais para sua chave de API e construir um relacionamento com a equipe para ajudar a melhorar seu produto.

Se o seu produto for aprovado, esperamos que você o mantenha em conformidade com todas as regras, políticas e leis aplicáveis. Cabe a você se manter atualizado sobre tudo isso. Se precisar de ajuda com isso, deixe-nos uma mensagem no Portal do Desenvolvedor ou entre no Desenvolvedor Discord.

Se sua proposta de produto for rejeitada, deixaremos uma mensagem no Portal do Desenvolvedor. Queremos que você tenha sucesso, então sinta-se à vontade para trabalhar conosco por meio desse sistema de mensagens para abordar nossas preocupações e colocar sua ótima ideia nas mãos dos jogadores.

Ao analisar os aplicativos, analisamos muitos fatores para determinar se é algo que beneficiará o ecossistema da Riot Games. Vale ressaltar que queremos ter certeza de que o produto não viole nenhuma lei ou política existente. Também queremos ter certeza de que o produto ajude os jogadores de alguma forma mensurável. Queremos ver produtos que ajudem os jogadores a melhorar em nossos jogos ou a acompanhar seu crescimento. O que não queremos ver são produtos que resolvam nossos jogos ou tornem tudo muito simples. Além disso, buscamos qualidade. Se o seu site não estiver completo, é improvável que aprovemos seu produto. Se você tiver dúvidas, basta entrar em contato pelo sistema de mensagens do Portal do Desenvolvedor.

Sobre esse sistema de mensagens...

Mensagens
Enviaremos notificações de aprovação e rejeição diretamente para você em suas mensagens de inscrição no Portal do desenvolvedor. Se tivermos dúvidas sobre uma inscrição pendente, garantiremos que você tenha uma maneira de nos responder no Portal do Desenvolvedor.

Se você tiver dúvidas específicas sobre o registro do seu produto, pode enviar uma mensagem em nosso Site de suporte para obter algum feedback. Usar o Site de Suporte é a maneira mais segura e rápida de obter uma resposta da equipe de Relações com Desenvolvedores. Além disso, se você tiver alguma dúvida sobre seus produtos ativos ou qualquer outra coisa no ecossistema, você pode enviar uma mensagem e nós responderemos. Uma ressalva: não use o sistema de mensagens de forma irresponsável. Este não é um canal para você conversar casualmente com os Rioters. Este é um canal de negócios e deve ser tratado como tal.

APIs da Web
Seu produto pode depender da API da Riot Games. Se isso acontecer, você precisará ter certeza de que entende alguns conceitos básicos sobre sua chave de API da Riot Games.

Chaves de API
Qualquer pessoa que entrar no portal do desenvolvedor receberá automaticamente uma chave de API que será associada à sua conta. Sua chave de API permite que você comece a usar a API imediatamente e nos permite gerar métricas de uso de API para sua chave. Ao todo, gerenciamos vários tipos de chaves de API. Veja as diferenças abaixo.

CHAVES DE API DE DESENVOLVIMENTO
A chave de API que foi gerada para você quando você entrou no portal do desenvolvedor é a API de desenvolvimento chave. Essas chaves de API provisórias são concedidas temporariamente para produtos que não são destinados ao consumo público, mas se beneficiam de acesso temporário à API. O objetivo de uma chave de API de desenvolvimento é que você mexa na API da Riot Games e potencialmente desenvolva um protótipo para um produto que você possa disponibilizar para uso da comunidade. Eles também são desativados a cada 24 horas. Você precisará redefinir o seu regularmente para mantê-lo ativo.

CHAVES DE API PESSOAIS
Você pode solicitar um API pessoal chave registrando seu produto. Chaves de API pessoais devem ser usadas para produtos destinados apenas ao desenvolvedor ou a uma pequena comunidade privada. Esses produtos podem ser registrados sem o processo de verificação, mas não serão aprovados para aumentos de limites de tarifas. Você pode solicitar acesso às APIs Padrão, mas não à API de Torneios. As chaves pessoais exigem uma descrição detalhada do produto.

Os usos aceitáveis para uma chave de API pessoal incluem:

bots para sites de streaming, placas, servidores de comunicação de voz, etc.
para exibir suas próprias estatísticas pessoais para seu site pessoal
projetos pessoais para reunir suas próprias estatísticas
pesquisa pessoal
projetos destinados ao uso pessoal e não à produção
O limite de tarifa para chaves pessoais é, por definição, muito limitado:

20 solicitações a cada 1 segundo
100 solicitações a cada 2 minutos
Observe que os limites de taxas são aplicados por região. Por exemplo, com o limite de taxa acima, você pode fazer 20 solicitações a cada 1 segundo para os endpoints NA e EUW League of Legends simultaneamente.

Você não pode executar seu aplicativo para consumo público usando uma chave pessoal, independentemente de quanto tempo leva o processo de aprovação de sua chave de produção. Observe que o consumo público inclui testes alfa/beta abertos.

CHAVES DE API DE PRODUÇÃO
Se você desenvolveu um aplicativo usando a API da Riot Games que gostaria de abrir para os jogadores, você deve solicitar um API de produção chave. Você não pode manter um produto público com uma chave de API de desenvolvimento. As chaves da API de produção têm um limite de taxa muito mais alto, adequado para sustentar o tráfego de um produto público.

As chaves da API de produção devem ser usadas para produtos destinados a grandes comunidades ou à Internet como um todo. Você pode solicitar acesso às APIs Padrão e à API de Torneios. Normalmente requer um protótipo funcional antes de receber uma chave de API.

O limite de taxa inicial de uma chave de API de produção é muito maior do que a chave de desenvolvimento:

500 solicitações a cada 10 segundos
30.000 solicitações a cada 10 minutos
Lembre-se de que esse limite de taxa é aplicado por região.

Para solicitar uma chave de produção com um limite de taxa expandido, clique em "Registrar projeto" no seu painel. O processo e o tempo necessário para obter uma chave de produção aprovada podem variar dependendo do seu projeto e da(s) região(s) de destino do aplicativo.

Observe que o limite padrão da taxa de produção atenderá às necessidades da grande maioria dos desenvolvedores, mas pode ser expandido se o desenvolvedor estiver em situação regular, tiver demonstrado um forte benefício para a comunidade e tiver superado constantemente o limite padrão de produção.

Se estiver trabalhando em vários projetos, você deve registrar cada um separadamente e cada um precisa ser aprovado individualmente para uma chave de API de produção separada.

SEGURANÇA DA CHAVE API
Como nota final sobre chaves de API, é importante mencionar que sua chave provavelmente acabará revogada se não estiver devidamente protegida. Proteger sua chave de API é um requisito para publicar um projeto conforme descrito no Políticas Gerais. Se determinarmos que sua chave não está protegida adequadamente, tomaremos medidas para protegê-la para você. :)

Códigos de resposta
A API da Riot Games retorna todos os dados em JSON válido. Algumas linguagens de programação incluem suporte nativo para JSON. Para quem não o faz, você pode encontrar uma biblioteca adequada em https://www.json.org.

Observe que nossas APIs retornam apenas valores não vazios para economizar largura de banda. Zero é considerado um valor vazio, assim como strings vazias, listas vazias e nulos. Qualquer campo numérico que não seja retornado pode ser considerado 0 (ou nulo, como preferir). Qualquer campo de lista que não seja retornado pode ser considerado uma lista vazia ou nula. Qualquer campo String que não seja retornado pode ser considerado uma string vazia ou nula.

CÓDIGOS DE RESPOSTA 2XX
Para 200 códigos de resposta, você sempre pode esperar o corpo da resposta documentado na página de referência da API. Apenas 200 códigos de resposta têm garantia de retornar um corpo de resposta como JSON.

Para códigos de resposta não 200, esteja ciente do seguinte:

Não há garantia de que um corpo de resposta será devolvido.

Se houver um corpo de resposta, não há garantia de que seja JSON.
Atualmente, retornamos JSON com informações de depuração legíveis por humanos, mas a estrutura e o conteúdo dessas informações de depuração estão sujeitos a alterações. Como exemplo...
    {
        "status": {
            "message": "Unauthorized",
            "status_code": 401,
        }
    }
O conteúdo de status, mensagem, e código_de status não há garantia de que sempre existam ou permaneçam constantes para um determinado código de resposta. 4. A lógica dentro do seu aplicativo deve falhar graciosamente com base apenas no código de resposta e não deve depender do corpo da resposta.

CÓDIGOS DE ERRO 4XX
A classe 4xx de códigos de erro tem como objetivo indicar que o cliente não forneceu uma solicitação válida. Abaixo estão as classes 4xx de códigos de erro mais comuns que você pode encontrar ao usar a API.

400 (Solicitação inválida) Este erro indica que há um erro de sintaxe na solicitação e, portanto, a solicitação foi negada. O cliente não deve continuar a fazer solicitações semelhantes sem modificar a sintaxe ou as solicitações que estão sendo feitas.

Razões comuns

Um parâmetro fornecido está no formato errado (por exemplo, uma string em vez de um número inteiro).
Um parâmetro fornecido é inválido (por exemplo, beginTime e startTime especificam um intervalo de tempo muito grande).
Um parâmetro obrigatório não foi fornecido.
401 (Não autorizado) Esse erro indica que a solicitação feita não continha as credenciais de autenticação necessárias (por exemplo, uma chave de API) e, portanto, o acesso foi negado ao cliente. O cliente não deve continuar a fazer solicitações semelhantes sem incluir uma chave de API na solicitação.

Razões comuns

Uma chave de API não foi incluída na solicitação.
403 (Proibido) Este erro indica que o servidor entendeu a solicitação, mas se recusa a autorizá-la. Não há distinção entre um caminho inválido ou credenciais de autorização inválidas (por exemplo, uma chave de API). O cliente não deve continuar a fazer solicitações semelhantes.

Razões comuns

Uma chave de API inválida foi fornecida com a solicitação de API.
Uma chave de API na lista negra foi fornecida com a solicitação de API.
A solicitação da API era para um caminho incorreto ou não suportado.
404 (Não encontrado) Este erro indica que o servidor não encontrou uma correspondência para a solicitação de API que está sendo feita. Não há indicação se a condição é temporária ou permanente.

Razões comuns

O ID ou nome fornecido não corresponde a nenhum recurso existente (por exemplo, não há nenhum Invocador que corresponda ao ID especificado).
Não existem recursos que correspondam aos parâmetros especificados.
415 (Tipo de mídia não suportado) Este erro indica que o servidor está se recusando a atender a solicitação porque o corpo da solicitação está em um formato que não é suportado.

Razões comuns

O Tipo de conteúdo o cabeçalho não foi definido adequadamente.
429 (Limite de taxa excedido) Este erro indica que o aplicativo esgotou o número máximo de chamadas de API permitidas por um determinado período. Se o cliente receber uma resposta de Limite de Taxa Excedido, o cliente deverá processar essa resposta e interromper futuras chamadas de API pela duração, em segundos, indicada pelo Tentar novamente depois cabeçalho. Os aplicativos que violam esta política podem ter seu acesso desativado para preservar a integridade da API. Consulte nossa documentação sobre limitação de tarifas abaixo para obter mais informações sobre como determinar se você teve uma limitação de tarifas e como evitá-la.

Razões comuns

Chamadas de API não regulamentadas.
CÓDIGOS DE ERRO 5XX
A classe 5xx ou códigos de erro indicam que o servidor está ciente de que cometeu um erro ou é incapaz de executar a solicitação. Abaixo estão as classes 5xx de códigos de erro mais comuns que você pode encontrar ao usar a API.

500 (Erro interno do servidor) Este erro indica uma condição ou exceção inesperada que impediu o servidor de atender a uma solicitação de API.

503 (Serviço indisponível) Este erro indica que o servidor não está disponível para lidar com solicitações devido a um motivo desconhecido. A resposta Serviço Indisponível implica uma condição temporária que será aliviada após algum atraso.

Limitação de taxa
Para controlar o uso da API da Riot Games, definimos limites de quantas vezes os endpoints podem ser acessados dentro de um determinado período de tempo. Esses limites são colocados em prática para minimizar abusos, manter um alto nível de estabilidade e proteger os sistemas subjacentes que impedem que a API fique sobrecarregada. Os sistemas subjacentes são os mesmos que alimentam nossos jogos, portanto, se estiverem sobrecarregados, a experiência do jogador sofre, e nossa primeira prioridade é proteger essa experiência.

TIPOS DE LIMITAÇÃO DE TAXA
Existem três tipos de limites usados na infraestrutura da API: limites de aplicativo, método e taxa de serviço.

Limites de taxa de aplicação
O primeiro tipo de limite é aplicado por chave de API e é chamado de limite de taxa de aplicação. Os limites de taxa de aplicativos são aplicados por região. Cada chamada feita para qualquer endpoint da API da Riot Games em uma determinada região conta contra o limite de taxa do aplicativo para essa chave nessa região. Por exemplo, chamadas para a API de dados estáticos não contam para o limite de taxa do aplicativo.

Limites de taxa do método
O segundo tipo de limite é imposto por ponto final (ou "método") para uma determinada chave de API e é chamado de limite de taxa do método. Limites de taxa de método também são aplicados por região. Cada chamada feita para qualquer endpoint da API da Riot Games em uma determinada região conta contra o limite de taxa de método para o método e a chave de API fornecidos nessa região.

Limites de taxa de serviço
O terceiro tipo de limite é aplicado por serviço e é chamado de limite de taxa de serviço. Limites de tarifas de serviço também são aplicados por região. Cada chamada feita para qualquer endpoint para um determinado serviço da API da Riot Games em uma determinada região conta com o limite de taxa de serviço para esse serviço naquela região. Quando os limites de taxa de serviço forem aplicados, nós os documentaremos, incluindo quais endpoints fazem parte do serviço de taxa limitada.

Não confunda limites de taxa de método com limites de taxa de serviço. Os limites de taxa do método aplicam-se individualmente a cada aplicação. Os limites de taxa de serviço se aplicam ao serviço e são compartilhados por todos os aplicativos que fazem chamadas para um serviço.

Outros Limites
Esses limites impostos pela infraestrutura da API não são os únicos gateways para os dados fornecidos. Alguns dos serviços subjacentes para determinados endpoints também podem implementar seus próprios limites de taxa, independentemente da infraestrutura da API. Nesses casos, você receberá uma resposta de erro 429, mas não haverá Tipo de limite de taxa X cabeçalho incluído na resposta. Somente quando a limitação de taxa for aplicada pela infraestrutura de borda da API esse cabeçalho será incluído.

Embora seja nossa política não revelar os detalhes de como nossa limitação de taxa funciona, você pode presumir, para os propósitos do seu código, que o bucket começa quando você faz sua primeira chamada de API.

Controle de versão
A partir de 2019, essas diretrizes de versionamento se aplicam a todas as APIs da Riot Games. No entanto, isso pode mudar no futuro. Certifique-se de verificar a documentação de cada jogo para o qual você está desenvolvendo produtos para ter certeza de que possui as informações mais recentes.

Atualmente, todas as APIs do League of Legends são da versão 4. Não incluímos mais uma versão secundária no caminho da API.

Depreciação
A partir de 2019, essas diretrizes de descontinuação se aplicam a todas as APIs da Riot Games. No entanto, isso pode mudar no futuro. Certifique-se de verificar a documentação de cada jogo para o qual você está desenvolvendo produtos para ter certeza de que possui as informações mais recentes.

Sempre que descontinuamos ou fazemos alterações em uma API, pretendemos oferecer suporte a versões antigas e novas da API por 60 dias. Após esse período, a versão antiga é descontinuada. Às vezes, as circunstâncias exigem que os períodos de depreciação sejam maiores ou menores que 60 dias. Os períodos de descontinuação serão comunicados aos desenvolvedores por meio das mídias sociais e do Developer Relations blog.

Os desenvolvedores devem ter cuidado ao desenvolver produtos com a API da Riot Games. Se uma API for descontinuada em favor de uma versão atualizada, a versão anterior não estará mais disponível. Quaisquer chamadas de uma chave de API válida para um endpoint obsoleto resultarão em um código de erro.
